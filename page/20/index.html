算法-散列表

Posted on 2021-07-12 Edited on 2022-06-27

散列表

散列表（Hash table，也叫哈希表）

哈希函数（映射）

根据Key来计算存储位置的计算规则我们称之为哈希函数

假如我们有一组数据,某位工程师每年的收入情况

直接定址法

Hash（Key）= A*Key + B。
优点：简单、均匀
缺点：需要事先知道关键字的分布情况(判断其对应关系是否为线性)

除留余数法

Hash( key ) = key % p ( p <= m)

m为地址数

p为一个不大于m，但最接近或者等于m的质数

散列表的运作流程

输入要查找的值key，使用 Hash(key) 得到一个值i

若 array[i] = key ，则查找成功

若 array[i] 不存在，则插入元素，返回 ”查找失败“

若 array[i] ≠ key，则进行哈希冲突处理

哈希冲突

哈希函数算出来的位置已有元素, 即为冲突

冲突处理

线性探查法

产生冲突，依次查看其后的下一个桶，如果发现空位置插入新元素

平方探测法(二次探测法)【+】

相对比上面的i+1, 平方探测法为 i+(平方序列)

平方序列: 

1, -1, 4, -4, 9, -9

线性探测容易聚集，二次探测聚集情况较线性探测要好

散列表长度是某个4k＋3形式的素数时，平方探测法就可以探查到整个散列表空间

插入20

h(20)=8冲突

+1冲突，-1冲突

+4 (9+4)%11=2 无冲突‘

冲突次数=3

链地址法

开散列法又叫链地址法(开链法)

载荷因子（装填因子、装载因子）

满1

散列表的载荷因子：a = 插入元素个数n / 散列表的长度m

a是散列表装满程度的标志因子。

散列表的平均查找长度依赖于散列表的装填因子

而不直接依赖于n和m

装填因子与平均查找长度

装填因子越大，散列表越满，越容易发生冲突

装填因子越小，越不容易发生冲突，平均查找长度就越小

散列表不满，容易查找不成功，但是其查找长度就是短了呀

（array[i] 若不存在则插入并返回”查找失败“）

散列表的效率

影响因素

散列函数、处理冲突的方法、装填因子

算法-插入排序

Posted on 2021-07-11 Edited on 2022-06-27

核心思想

直接插入排序

核心思想

新建子序列，不断挪位置, 然后最后把目标元素插入正确位置

Ps :

这和冒泡排序有所不同,冒泡排序是一路对换位置, 换上去

而直接插入排序是先挪好位置然后把目标元素插入

(当前代码的排序规则是从小到大)

对应已经排好序的数列，插入排序的时间复杂度是O(n)

只比较一次即插入

折半插入排序

相对比上面的边比较边移动，这里是利用二分法先找到合适位置，然后再统一移动

这样做的话，不需要每一个都进行比较O(x^2^)而是O(nlog2n)

但是这仅仅是比较次数减少，其移动的次数没有减少

故折半插入排序的时间复杂度仍为O(n^2^)

while的条件不可改

直接看代码的话，这里的逻辑会把人绕晕

需要明确一下方向

先看while循环内部

①当要插入的元素的值小于mid的值（这里的while没有主动跳出语句，故最后都会收敛于low=high）

则应该插入到mid的左边，即mid-1处

转换一下思维，相当于把mid右挪，然后把目标元素插入到空出的位置

然后看第二个for循环（用于挪元素）

结合while内的high=mid-1和 j >=high+1，表示即把原mid给右挪了（这里的+1花点时间理解一下就懂了）

最后插入目标元素到空出的位置即可

间接实现了插入到mid左边

②当要插入的元素的值大于mid的值

则插入到mid+1处

结合while内的high不变和 j >=high+1，表示即把原mid+1给右挪了（和上面区分开来）

然后把目标元素插入到mid+1处

希尔排序（该方法比较抽象，注意看解析）

这是一个分组插入排序的算法

分组规则：根据步长将不相邻的n个元素构成一个分组

然后对各个分组进行插入排序，即相对于上面直接插入排序，这里交换位置的规则是

而直接插入排序的交换规则是

步长的变换规律：o(log2n)

直到步长为1时

即为直接插入排序

不直接进行插入排序的原因

直接插入排序对于已经排好序的数列时间复杂度是O(n)

最坏是O(n^2^)

所以这里采用希尔分组先对数列进行一个大概的排序（可以理解为预处理）

“大概的排序”，即上面说的有步长间隔的插入排序

使其基本有序后，再进行直接插入排序，则可以提高直接插入排序的效率

希尔排序的时间复杂度较为复杂

大概知道就好，一般为O(n^1.3^)

最坏为O(n^2^)
